  • SessionBuilder

    Session builder is responsible for setting up encrypted sessions. Once a session has been established, session_cipher can be used to encrypt/decrypt messages in that session.

    Sessions are built from one these different possible vectors:

    • A session_pre_key_bundle retrieved from a server
    • A pre_key_signal_message received from a client

    Sessions are constructed per Signal Protocol address (recipient name + device ID tuple). Remote logical users are identified by their recipient name, and each logical recipient can have multiple physical devices.
